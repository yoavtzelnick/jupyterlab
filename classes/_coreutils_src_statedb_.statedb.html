fetch

  • fetch(id: string): Promise<ReadonlyJSONValue | undefined>
    • Defined in coreutils/src/statedb.ts:172

    Retrieve a saved bundle from the database.

    Parameters

    • id: string

      The identifier used to retrieve a data bundle.

    Returns Promise<ReadonlyJSONValue | undefined>

    A promise that bears a data payload if available.

    Notes

    The id values of stored items in the state database are formatted: 'namespace:identifier', which is the same convention that command identifiers in JupyterLab use as well. While this is not a technical requirement for fetch(), remove(), and save(), it is necessary for using the fetchNamespace() method.

    The promise returned by this method may be rejected if an error occurs in retrieving the data. Non-existence of an id will succeed with null.

fetchNamespace

  • fetchNamespace(namespace: string): Promise<IStateItem[]>

  • Implementation of IStateDB.fetchNamespace

    • Defined in coreutils/src/statedb.ts:192

    Retrieve all the saved bundles for a namespace.

    Parameters

    • namespace: string

      The namespace to retrieve.

    Returns Promise<IStateItem[]>

    A promise that bears a collection of payloads for a namespace.

    Notes

    Namespaces are entirely conventional entities. The id values of stored items in the state database are formatted: 'namespace:identifier', which is the same convention that command identifiers in JupyterLab use as well.

    If there are any errors in retrieving the data, they will be logged to the console in order to optimistically return any extant data without failing. This promise will always succeed.
